## Architecture Decision Records (ADRs)

ADRs live in `docs/adr/` as `ADR-NNN-<slug>.md`, following a fixed format:
Status → Context and Problem Statement → Decision Drivers → Considered Options →
Decision Outcome → Pros/Cons → (optional) Related.

The `Proposed` → `Accepted`-on-merge lifecycle is **mandatory** and defined in
the shared guide — see "Architecture Decision Records (ADRs) — MANDATORY" there.
